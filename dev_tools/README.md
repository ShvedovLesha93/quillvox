# Development Tools

## Translation Management

This project uses GNU gettext for internationalization (i18n) of Python code. Translation strings are marked with `_()` in the codebase and managed.

### Available Scripts

#### 1. Extract translatable strings
```bash
./i18n-extract.py
```
Scans all Python files in the `app/` directory for strings marked with `_()` and generates `app/locales/messages.pot` (translation template).

#### 2. Initialize or update language files
```bash
./i18n-update.py ru pl de
```
- **For new languages**: Creates `app/locales/{lang}/LC_MESSAGES/messages.po`
- **For existing languages**: Merges new strings from `messages.pot` into existing `.po` files
- Accepts one or more language codes (e.g., `ru`, `pl`, `de`, `fr`)

**Note**: Don't create translations for English if your source code is already in English.

#### 3. Compile translations
```bash
./i18n-compile.py ru pl de
```
Compiles `.po` files into binary `.mo` files that the application uses at runtime. Language codes must match those used in step 2.

### Typical Workflow

1. **Extract** new/updated strings from code:
```bash
   ./i18n-extract.py
```

2. **Update** translation files for your target languages:
```bash
   ./i18n-update.py ru pl
```

3. **Translate** the strings by editing `app/locales/{lang}/LC_MESSAGES/messages.po` files with a text editor or translation tool like [Poedit](https://poedit.net/)

4. **Compile** the translations:
```bash
   ./i18n-compile.py ru pl
```

5. Test your application with the new translations

### File Structure
```
app/
├── locales/
│   ├── messages.pot          # Template (generated by extract)
│   ├── ru/
│   │   └── LC_MESSAGES/
│   │       ├── messages.po   # Russian translations (human-editable)
│   │       └── messages.mo   # Compiled binary (generated)
│   └── pl/
│       └── LC_MESSAGES/
│           ├── messages.po   # Polish translations
│           └── messages.mo
└── [your Python files]
```
