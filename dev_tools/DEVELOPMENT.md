# Development Tools

## Translation Management

This project uses GNU gettext for internationalization (i18n) of Python code.
Translation strings are marked with `_()` in the codebase and managed.

### Available Scripts

#### 1. Extract translatable strings

```bash
./i18n-extract.py
```

Scans all Python files in the `app/` directory for strings marked with `_()`
and generates `app/locales/messages.pot` (translation template).

#### 2. Initialize or update language files

```bash
./i18n-update.py ru pl de
```

- **For new languages**: Creates `app/locales/{lang}/LC_MESSAGES/messages.po`
- **For existing languages**: Merges new strings from `messages.pot` into existing `.po` files
- Accepts one or more language codes (e.g., `ru`, `pl`, `de`, `fr`)

**Note**: Don't create translations for English if your source code
is already in English.

#### 3. Compile translations

```bash
./i18n-compile.py ru pl de
```

Compiles `.po` files into binary `.mo` files that the application uses at runtime.
Language codes must match those used in step 2.

### Typical Workflow

1. **Extract** new/updated strings from code:

    ```bash
    ./i18n-extract.py
    ```

2. **Update** translation files for your target languages:

    ```bash
      ./i18n-update.py ru pl
    ```

3. **Translate** the strings by editing `app/locales/{lang}/LC_MESSAGES/messages.po`
files with a text editor or translation tool like [Poedit](https://poedit.net/)

4. **Compile** the translations:

    ```bash
    ./i18n-compile.py ru pl
    ```

5. Test your application with the new translations

### File Structure

```bash
app/
├── locales/
│   ├── messages.pot          # Template (generated by extract)
│   ├── ru/
│   │   └── LC_MESSAGES/
│   │       ├── messages.po   # Russian translations (human-editable)
│   │       └── messages.mo   # Compiled binary (generated)
│   └── pl/
│       └── LC_MESSAGES/
│           ├── messages.po   # Polish translations
│           └── messages.mo
└── [your Python files]
```

## Icon Management

This project uses SVG icons with theme-specific variants compiled into Qt resources.

### Icon Structure

Icons are organized by theme in the `app/resources/icons/` directory:

```bash
app/resources/icons/
├── dark_theme/
│   ├── play.svg
│   ├── pause.svg
│   └── ...
└── light_theme/
    ├── play.svg
    ├── pause.svg
    └── ...
```

**Important**: Each icon must exist in both `dark_theme/` and `light_theme/` folders with identical
filenames. The SVG content differs only in color values to match the respective theme.

### Adding New Icons

1. **Prepare SVG files** with appropriate colors for each theme
2. **Place files** in both theme folders with matching names:
   - `app/resources/icons/light_theme/play.svg`
   - `app/resources/icons/dark_theme/play.svg`

3. **Generate icon literals** (for IDE autocomplete):

   ```bash
   python dev_tools/generate_icon_literal.py
   ```

   This updates type hints in `app/views/ui_utils/icons.py` to provide autocomplete when assigning
   icons in code.

4. **Compile resources**:

   ```bash
   python dev_tools/compile_resources.py
   ```

   This updates type hints in `app/views/ui_utils/icons.py` to provide autocomplete when assigning
   icons in code. It also updates `app/resources/resources.qrc` with the icon file references.

## Dev Reloader

A lightweight development tool for running and reloading the app

### Usage

```bash
uv run dev_tools/dev_reloader.py <module>
```

### Example

```bash
uv run dev_tools/dev_reloader.py app.main
```

### Controls

| Key                        | Action             |
|----------------------------|--------------------|
| `Ctrl-C`                   | Restart the module |
| `Ctrl-C` twice (within 2s) | Stop the runner    |

The runner will also exit automatically if the module crashes, propagating the exit code.

## Troubleshooting

If you encounter the error `Failed to canonicalize script path` when running `compile_resources.py`,
try removing `.venv` and reinstalling dependencies:

```bash
rm -rf .venv  # or rmdir /s .venv on Windows
uv sync

```

### Workflow Summary

1. Add SVG files to `dark_theme/` and `light_theme/` folders
2. Run `generate_icon_literal.py` to update autocomplete literals
3. Run `compile_resources.py` to compile icons into the resource file
4. Use the icons in your code with full autocomplete support
